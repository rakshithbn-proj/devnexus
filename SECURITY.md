# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report them by emailing the maintainer via the contact info on the [GitHub profile](https://github.com/rakshithbn-proj).

You will receive a response within 72 hours. If the issue is confirmed, a fix will be prioritised.

## Scope

- Credential leakage (PATs exposed in logs or written to disk insecurely)
- Authentication bypass
- Malicious server behaviour (e.g. exfiltrating credentials or request data)

## Out of scope

- Vulnerabilities in Jira/Bitbucket Server itself
- Issues requiring physical access to the machine
